# apps/demo

Первый рабочий demo panel СТРОИТ.РФ.

Назначение: дать возможность пройти базовый путь через браузер без добавления новых backend endpoints.

## Запуск

Backend должен быть запущен отдельно:

```bash
npm run backend:dev
```

Demo panel запускается отдельным локальным сервером:

```bash
npm run demo:dev
```

Открыть:

```text
http://localhost:3100
```

По умолчанию demo proxy отправляет `/api/*` на `http://localhost:3000`.

Можно переопределить:

```bash
DEMO_PORT=3101 DEMO_BACKEND_URL=http://localhost:3000 npm run demo:dev
```

## Что покрыто

- login через `POST /api/v1/auth/login`;
- хранение JWT в `localStorage`;
- профиль через `GET /api/v1/auth/me`;
- начало и завершение смены;
- получение задач монтажника через `GET /api/v1/tasks/my`;
- открытие задачи;
- получение этапов;
- старт и завершение этапа;
- загрузка фото через `multipart/form-data`;
- просмотр истории событий.

## Роли для полного demo path

Рабочий путь монтажника проходит под `WORKER`, если этому пользователю уже назначены задачи.

Для подготовки демо-данных можно войти под `CREATOR`, `DIRECTOR` или `FOREMAN` и создать демо-задачу через кнопку в интерфейсе. Это использует существующие `POST /api/v1/tasks` и `POST /api/v1/tasks/:taskId/steps`.

Для роли `WORKER` доступны смена, список своих задач, чтение конкретной задачи, чтение этапов, старт/завершение этапа и загрузка фото. История событий для чистого `WORKER` остается ограничением текущего RBAC.
