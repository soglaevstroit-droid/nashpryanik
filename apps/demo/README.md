# apps/demo

Первый рабочий demo panel СТРОИТ.РФ.

Назначение: дать возможность пройти базовый путь через браузер без добавления новых backend endpoints.

## Запуск

Backend должен быть запущен отдельно:

```bash
npm run backend:dev
```

Demo panel запускается отдельным локальным сервером:

```bash
npm run demo:dev
```

Открыть:

```text
http://localhost:3100
```

По умолчанию demo proxy отправляет `/api/*` на `http://localhost:3000`.

Можно переопределить:

```bash
DEMO_PORT=3101 DEMO_BACKEND_URL=http://localhost:3000 npm run demo:dev
```

## Что покрыто

- login через `POST /api/v1/auth/login`;
- хранение JWT в `localStorage`;
- профиль через `GET /api/v1/auth/me`;
- начало и завершение смены;
- получение задач;
- открытие задачи;
- получение этапов;
- старт и завершение этапа;
- загрузка фото через `multipart/form-data`;
- просмотр истории событий.

## Роли для полного demo path

Полный путь в текущем API удобнее проходить под `CREATOR`, `DIRECTOR` или `FOREMAN`, потому что этим ролям доступны список задач, создание демо-задачи, этапы, фото и события.

Для роли `WORKER` доступны смена, чтение конкретной задачи по id, чтение этапов конкретной задачи, старт/завершение этапа и загрузка фото, но недоступен общий список задач и история событий.
