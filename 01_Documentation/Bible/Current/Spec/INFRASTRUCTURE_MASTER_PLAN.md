# Назначение документа

Данный документ описывает полную инфраструктуру проекта СТРОИТ.РФ.

Его цель — обеспечить возможность развернуть проект с нуля без потери архитектурных решений.

Документ описывает:

- серверную инфраструктуру;
- сетевую архитектуру;
- доменную структуру;
- контейнеризацию;
- безопасность;
- мониторинг;
- резервное копирование;
- масштабирование;
- стратегию развития инфраструктуры.

---

# Главная философия

Инфраструктура должна быть:

- простой;
- надежной;
- масштабируемой;
- безопасной;
- документированной;
- автоматизированной.

Главный принцип:

> Любой новый инженер должен иметь возможность развернуть проект, используя только документацию.

# Этапы развития инфраструктуры

## Этап 1 — MVP

Количество сотрудников:

до 10

Количество объектов:

1

Инфраструктура:

Один VPS.

Все сервисы работают внутри Docker Compose.

Количество сотрудников:

до 100

Количество объектов:

до 10

- отдельный Backup Server;
- мониторинг;
- ежедневные автоматические резервные копии.

## Этап 3 — Рост

Количество сотрудников:

100–1000

Количество объектов:

десятки

- отдельная PostgreSQL;

- распределенное хранение файлов;

---

# Disaster Recovery

Если выходит из строя сервер:

1. Поднимается новый VPS.

2. Устанавливается Docker.

3. Загружается Backup.

4. Запускается Docker Compose.

5. Проверяется доступность системы.

Цель восстановления:

менее одного часа.

- время ответа API менее 300 мс.

- выделенный MinIO;

1000 пользователей

необходимо внедрить:

- Load Balancer;
- Kubernetes (при необходимости);
- CDN;

---

# Правила инфраструктуры

1. Все сервисы работают в Docker.

2. Production не изменяется вручную.

3. Все изменения проходят через Git.

4. Все конфигурации документируются.

5. Backup обязателен.

6. Мониторинг обязателен.

7. SSL обязателен.

8. Логи сохраняются.

9. Любая инфраструктура должна быть воспроизводимой.

10. Любая новая технология должна иметь инженерное обоснование.

---

# Definition of Done

Инфраструктура считается готовой если:

✓ сервер доступен;

✓ Docker работает;

✓ Backend запускается;

✓ PostgreSQL подключена;

✓ Redis подключен;

✓ MinIO доступен;

✓ SSL работает;

✓ домены открываются;

✓ Backup выполняется;

✓ документация актуальна.

---

# Заключение

Инфраструктура проекта должна развиваться постепенно.

Мы не строим систему для одного объекта.

Мы создаем платформу, которая должна одинаково уверенно работать как на одном объекте, так и в компании с тысячами сотрудников.

Каждое инфраструктурное решение должно учитывать не только сегодняшний день, но и будущее развитие проекта.
