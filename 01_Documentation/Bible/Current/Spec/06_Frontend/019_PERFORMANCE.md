# 019 — Performance

## Назначение
Документ описывает производительность Flutter-приложения.

## Lazy loading
Детальные данные загружаются при открытии экрана:
- карточка сотрудника;
- история активности;
- финансовый отчет;
- детализация AI-рекомендации.

## Pagination
Пагинация используется для:
- задач;
- сотрудников;
- уведомлений;
- истории;
- отчетных строк.

## Image cache
Фото и thumbnails кэшируются с учетом company scope и auth. Кэш не должен показывать фото после logout другому пользователю.

## Memory
Большие фото не держатся в памяти дольше необходимого. Compression и thumbnail generation должны освобождать исходные buffers.

## Rebuild optimization
State должен быть разделен так, чтобы обновление upload progress не перестраивало весь экран.

## Network
API client использует timeout, retry и cancellation для закрытых экранов.

## Метрики
Отслеживаются:
- время открытия стартового экрана;
- время загрузки списка задач;
- время upload фото;
- количество rebuild;
- memory pressure;
- crash-free sessions.
