# 016 — Role Based UI

## Назначение
Документ описывает различия интерфейса по ролям. Frontend показывает только те экраны и действия, которые разрешены ролью и подтверждены backend.

## Монтажник
Показывать:
- статус `На работе` / `Ушел`;
- баланс монет;
- таймер смены;
- общий список задач объекта;
- текущую задачу;
- этапы;
- фото-подтверждение;
- сообщения;
- профиль и историю задач.

Не показывать:
- тариф;
- лишнюю аналитику;
- финансовые отчеты;
- выплаты;
- управление задачами прораба.

## Прораб
Показывать:
- список сотрудников;
- статус сотрудника;
- текущую задачу;
- прогресс этапов;
- монеты и время;
- создание задач;
- назначение исполнителей;
- создание этапов;
- сообщения и черновики.

Не показывать:
- ручное начисление монет;
- утверждение выплат;
- финансовые решения.

## Финансист
Показывать:
- общие монеты;
- сотрудников;
- аналитику;
- AI-рекомендации;
- выплаты;
- историю выплат;
- финансовые отчеты;
- уведомления;
- настройки.

Не показывать:
- создание задач как у прораба;
- выполнение этапов как у монтажника;
- редактирование начисленных монет вручную.

## Администратор
Роль администратора не описана в текущих источниках истины `00_Project`, `004_USER_ROLES.md`, API Bible и Backend Bible. Frontend не должен реализовывать отдельный admin UI, маршруты или права до появления этой роли в фундаментальной документации.

## Руководитель
Руководительская роль описана как управленческая аналитика без отдельного набора MVP-экранов. Frontend может показывать только те аналитические данные, которые разрешены backend и описаны документацией.

## Правила
- Role берется из JWT/backend.
- Локальная роль не является источником истины.
- При смене роли или logout UI полностью пересобирает navigation tree.
- Protected routes проверяют роль до открытия экрана.
