# 012 — Offline Mode

## Назначение
Offline mode позволяет приложению сохранять рабочий контекст при временной потере сети и безопасно повторять допустимые действия.

## Что доступно offline
- просмотр последнего кэша задач;
- просмотр текущего экрана;
- подготовка фото к отправке;
- локальное сохранение upload task;
- просмотр локально сохраненных уведомлений.

## Что требует online
- login;
- refresh token;
- взятие задачи;
- финальное завершение этапа;
- начисление монет;
- создание выплаты;
- получение AI-рекомендации;
- выгрузка отчета.

## Очередь синхронизации
Очередь хранит операции:
- photo upload;
- статус уведомления;
- безопасный повтор подтверждения, если операция имеет idempotency key.

Финансовые действия требуют особой осторожности и не должны silently повторяться без обработки `PAYMENT_CONFLICT`.

## Конфликты
Конфликт возникает, если:
- задача уже взята другим сотрудником;
- у сотрудника уже активная задача;
- выплата уже создана;
- token устарел.

UI показывает conflict state и предлагает обновить данные.

## Повторная отправка
Повтор выполняется после восстановления сети. App сохраняет порядок операций, если они связаны одной задачей или фото.

## Принцип
Offline mode не меняет бизнес-логику и не делает локальные действия финальными без backend.
