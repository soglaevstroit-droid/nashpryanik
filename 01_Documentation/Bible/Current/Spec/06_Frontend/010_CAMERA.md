# 010 — Camera

## Назначение
Камера используется для фотофиксации ключевых действий: начало работы, выполнение этапа, невозможность выполнить задачу, завершение работы и фото-пример этапа.

## Разрешения
Перед открытием камеры приложение проверяет permission. Если доступ запрещен, открывается состояние, соответствующее `021_System_Camera_Error`.

## Camera flow
1. Пользователь открывает действие, требующее фото.
2. App проверяет permission.
3. Камера открывается в full-screen режиме.
4. Пользователь делает фото.
5. App показывает preview.
6. Пользователь подтверждает или делает повторное фото.
7. Фото передается в upload queue.

## Повторное фото
Пользователь может переснять фото до отправки. После успешной отправки фото становится частью истории и не заменяется без нового события.

## Качество
Фото должно быть достаточно качественным для проверки. App может применять сжатие, но не должен разрушать читаемость результата.

## Ошибки
- permission denied;
- camera unavailable;
- capture failed;
- file read failed.

Ошибки показываются понятным экраном без потери контекста задачи или этапа.

## Связь с API
После capture используется `POST /api/v1/photos` с `multipart/form-data`.

## Ограничения
Камера не создает бизнес-событие сама по себе. Бизнес-событие создается backend после привязки фото к действию.
