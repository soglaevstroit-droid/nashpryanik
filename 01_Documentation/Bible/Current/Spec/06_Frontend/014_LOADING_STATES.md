# 014 — Loading States

## Назначение
Документ описывает виды загрузки в приложении.

## Виды
- Initial loading.
- Pull to refresh.
- Inline loading.
- Skeleton.
- Saving.
- Uploading photo.
- Exporting report.
- Refreshing token.

## Skeleton
Skeleton используется для:
- списков задач;
- списков сотрудников;
- карточек аналитики;
- финансовых таблиц;
- истории.

Skeleton должен сохранять размер будущего контента и не ломать композицию.

## Progress
Progress используется для:
- загрузки фото;
- подготовки отчета;
- синхронизации offline queue.

## Retry
Retry показывается при:
- сетевой ошибке;
- ошибке загрузки фото;
- ошибке API;
- AI analysis error.

## Disabled action
Кнопка disabled, если:
- нет обязательного фото;
- форма невалидна;
- идет saving;
- роль не имеет права;
- действие недоступно по состоянию задачи или смены.

## Правило
Loading state не должен скрывать основную бизнес-логику и не должен менять итоговое состояние без ответа backend.
