# 015 — Design System Usage

## Назначение
Документ фиксирует применение существующего Design System во Flutter.

## Theme tokens
Flutter theme использует утвержденные цвета:
- Brand Orange `#F07A00`;
- Black `#111111`;
- White `#FFFFFF`;
- Light Background `#F7F7F7`;
- Card Border `#E5E7EB`;
- Text Gray `#6B7280`;
- Success Green `#22C55E`;
- Info Blue `#3B82F6`;
- Error Red `#EF4444`;
- Disabled Gray `#D1D5DB`.

## Правила
- Оранжевый только для главных действий и ключевых акцентов.
- Зеленый только для успеха и статуса `На работе`.
- Красный только для ошибок и опасных действий.
- Серый для недоступных задач и второстепенного текста.
- Один экран — одна главная задача.
- На экране одно главное действие.

## Components
Компоненты создаются как Flutter widgets, но визуально повторяют Design System и UI Bible.

## Запреты
- менять утвержденные цвета;
- создавать новые стили карточек;
- перегружать экран лишними блоками;
- менять нижнюю навигацию без UI Bible;
- показывать монтажнику тариф и лишнюю аналитику.

## Golden tests
Ключевые компоненты должны иметь golden tests, чтобы контролировать сохранение референсов минимум на 95%.
