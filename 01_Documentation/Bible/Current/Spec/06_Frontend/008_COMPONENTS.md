# 008 — Components

## Назначение
Документ описывает переиспользуемые Flutter-компоненты. Все компоненты используют существующий Design System без изменения цветов, радиусов и визуальной логики.

## Базовые компоненты
- `AppScaffold` — общий каркас экрана.
- `RoleBottomNavigation` — нижняя навигация роли.
- `PrimaryActionButton` — главное действие экрана.
- `SecondaryActionButton` — вторичное действие.
- `StatusBadge` — статус `На работе`, `Ушел`, задача, уведомление.
- `InfoCard` — мягкая карточка с данными.
- `MetricCard` — карточка финансового или рабочего показателя.
- `EmployeeListItem` — строка сотрудника.
- `TaskListItem` — строка задачи.
- `StepCard` — карточка этапа.
- `PhotoUploadBlock` — блок выбора, предпросмотра и загрузки фото.
- `EmptyStateView`;
- `ErrorStateView`;
- `LoadingSkeleton`;
- `RetryButton`.

## Финансовые компоненты
- `CoinsValue`;
- `WorkDurationValue`;
- `AIRecommendationCard`;
- `PaymentSummaryCard`;
- `FinanceFilterBar`;
- `ReportTable`.

## Правила компонента
- Компонент не содержит сетевой логики.
- Компонент получает готовые props/view model.
- Компонент не вычисляет бизнес-решение.
- Состояния Loading, Error, Empty должны сохранять композицию экрана.

## Design System
Используются:
- белый фон;
- мягкие карточки;
- `#F07A00` для главного действия;
- зеленый только для успеха и `На работе`;
- красный только для ошибок и опасных действий;
- серый для недоступного.

## Запрещено
- добавлять новый стиль карточек;
- менять нижнюю навигацию без UI Bible;
- показывать тариф монтажнику;
- делать AI-рекомендацию похожей на утвержденное решение.
