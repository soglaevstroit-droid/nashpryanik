# 009 — Forms

## Назначение
Документ описывает формы Flutter-приложения: ввод, валидацию, ошибки, фокус и маски.

## Типы форм
- login;
- создание задачи;
- создание этапа;
- причина `Невозможно выполнить`;
- комментарий финансиста;
- фильтры аналитики;
- настройки финансиста.

## Валидация
Frontend выполняет локальную валидацию для быстрых ошибок:
- обязательное поле;
- UUID route params;
- корректный период;
- неотрицательная сумма;
- поддержанный enum;
- обязательное фото, если сценарий требует.

Backend остается источником истины и повторно валидирует данные.

## Ошибки
Ошибки API `VALIDATION_ERROR` мапятся на поля формы. Общие ошибки показываются как error state или message block.

## Фокус
После ошибки фокус переводится на первое ошибочное поле. На mobile клавиатура не должна закрывать главное действие.

## Маски
Допустимые маски:
- телефон;
- дата;
- сумма монет или премии;
- период.

## Сохранение
Форма переходит в `Saving`. Кнопка главного действия блокируется до ответа backend.

## Запрещено
- сохранять финансовое решение без явного нажатия финансиста;
- подставлять AI-рекомендацию как утвержденную премию без действия;
- считать локальную валидацию финальной проверкой.
