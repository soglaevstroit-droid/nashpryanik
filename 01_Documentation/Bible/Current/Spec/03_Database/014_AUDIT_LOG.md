# 014 — Audit Log

## Назначение
Журнал `audit_log` фиксирует значимые действия системы. Он нужен для прозрачности, восстановления истории, проверки финансовых решений и разбора спорных ситуаций.

## Таблица `audit_log`

| Поле | Тип | Обяз. | Ограничения | Описание |
|---|---|---:|---|---|
| `id` | `uuid` | да | PK | Идентификатор записи. |
| `company_id` | `uuid` | да | FK `companies.id` | Компания. |
| `actor_user_id` | `uuid` | нет | FK `users.id` | Пользователь, если действие пользовательское. |
| `actor_role` | `user_role` | нет | enum | Роль на момент действия. |
| `event_type` | `text` | да | length > 0 | Тип события. |
| `entity_type` | `text` | да | length > 0 | Тип сущности. |
| `entity_id` | `uuid` | да |  | Идентификатор сущности. |
| `old_values` | `jsonb` | нет | nullable | Предыдущее состояние. |
| `new_values` | `jsonb` | нет | nullable | Новое состояние. |
| `metadata` | `jsonb` | нет | nullable | Контекст события. |
| `created_at` | `timestamptz` | да | default now() | Время события. |

## Что логируется
- вход и выход пользователя;
- начало и завершение смены;
- взятие задачи;
- создание задачи;
- назначение исполнителей;
- создание и изменение этапов;
- загрузка фото;
- завершение этапа;
- невозможность выполнить этап;
- завершение задачи;
- начисление монет;
- создание AI-рекомендации;
- просмотр AI-рекомендации;
- создание и обработка уведомления;
- создание выплаты;
- изменение настроек финансиста;
- ошибки конфликтов, влияющие на бизнес-состояние.

## Срок хранения
Записи аудита хранятся столько же, сколько требуется для восстановления истории задач, смен, фото, монет, рекомендаций и выплат. Для активной бизнес-истории аудит не удаляется физически.

## Восстановление истории
История восстанавливается через:
- последовательность событий `audit_log`;
- доменные таблицы;
- связи фото;
- транзакции монет;
- AI-рекомендации;
- уведомления;
- финансовые записи.

## Индексы
- `idx_audit_company_created` на `(company_id, created_at desc)`.
- `idx_audit_entity` на `(company_id, entity_type, entity_id, created_at desc)`.
- `idx_audit_actor` на `(company_id, actor_user_id, created_at desc)` where `actor_user_id is not null`.
- `idx_audit_event_type` на `(company_id, event_type, created_at desc)`.

## Ограничения
- Аудит не должен содержать бинарные файлы фото.
- Аудит не заменяет доменные таблицы.
- Финансовые решения должны иметь доменную запись и audit-запись.
- AI-события должны фиксировать, что ИИ рекомендовал, а человек утверждал.
