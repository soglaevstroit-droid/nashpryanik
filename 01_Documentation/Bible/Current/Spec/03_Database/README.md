# 03 Database

## Назначение
Раздел `03_Database` описывает PostgreSQL-модель НАШПРЯНИК.РФ. Документация фиксирует сущности, таблицы, связи, enum, аудит, индексы и правила масштабирования базы данных.

База данных должна поддерживать текущую бизнес-логику UI Bible и Architecture Bible: роли, объекты, задачи, этапы, смены, фото, монеты, уведомления, AI-рекомендации, выплаты, историю и настройки без добавления новых бизнес-сценариев.

## Документы раздела
- `001_DATABASE_OVERVIEW.md` — общая архитектура хранения данных.
- `002_ENTITIES.md` — основные сущности и ключевые поля.
- `003_RELATIONSHIPS.md` — связи, обязательность и правила удаления.
- `004_TABLE_USERS.md` — пользователи, роли и привязки.
- `005_TABLE_OBJECTS.md` — компании и строительные объекты.
- `006_TABLE_TASKS.md` — задачи и исполнители.
- `007_TABLE_TASK_STEPS.md` — этапы задач.
- `008_TABLE_WORK_SESSIONS.md` — рабочие смены.
- `009_TABLE_PHOTOS.md` — метаданные фотографий и связь со storage.
- `010_TABLE_COINS.md` — транзакции монет.
- `011_TABLE_NOTIFICATIONS.md` — уведомления.
- `012_TABLE_AI_RECOMMENDATIONS.md` — AI-рекомендации.
- `013_ENUMS.md` — перечисления.
- `014_AUDIT_LOG.md` — журнал действий.
- `015_INDEXING.md` — стратегия индексации.
- `016_SCALABILITY.md` — масштабирование базы данных.

## Базовые правила
- PostgreSQL является основной структурированной базой данных.
- Фотографии хранятся в файловом хранилище, PostgreSQL хранит метаданные и ссылки.
- Каждая доменная таблица готова к работе с несколькими компаниями через `company_id`.
- Исторические записи не удаляются физически без отдельного архивного процесса.
- Ролевые ограничения проверяются backend, а база поддерживает их связями и индексами.
- ИИ рекомендует, человек утверждает.
