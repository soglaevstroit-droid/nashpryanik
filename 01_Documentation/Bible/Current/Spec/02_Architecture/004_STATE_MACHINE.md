# 004 — State Machine

## Назначение
Документ описывает состояния ключевых сущностей системы и допустимые переходы между ними. Состояния должны использоваться согласованно в UI, API, backend, аналитике и тестах.

## Пользователь

### Состояния
- Не авторизован.
- Авторизован.
- Активен.
- Офлайн.
- Доступ запрещен.

### Переходы
- Не авторизован → Авторизован: успешный вход.
- Авторизован → Активен: открытие стартового экрана роли.
- Активен → Офлайн: потеря сети.
- Офлайн → Активен: восстановление сети и синхронизация.
- Любое состояние → Доступ запрещен: попытка действия без роли или прав.
- Авторизован → Не авторизован: выход или истечение сессии.

## Рабочая смена

### Состояния
- Нет активной смены.
- Начало подтверждается.
- На работе.
- Завершение подтверждается.
- Ушел.

### Переходы
- Нет активной смены → Начало подтверждается: монтажник нажал начало работы.
- Начало подтверждается → На работе: подтверждение выполнено, фото загружено, если требуется экраном.
- На работе → Завершение подтверждается: монтажник нажал завершение работы.
- Завершение подтверждается → Ушел: смена закрыта.
- Ушел → Начало подтверждается: новая смена.

### Ограничения
- Монеты начисляются только в состоянии `На работе`.
- Если подтверждение начала или завершения не удалось, состояние не должно считаться финальным.

## Задача

### Состояния
- Свободная.
- Назначенная.
- В работе.
- Завершенная.
- Недоступная.
- Отмененная, если статус используется реализацией.

### Переходы
- Свободная → В работе: монтажник взял задачу.
- Назначенная → В работе: назначенный исполнитель взял задачу.
- В работе → Завершенная: выполнены необходимые этапы и подтверждения.
- Свободная или Назначенная → Недоступная: пользователь не имеет права взять задачу или уже выполняет другую.
- Любое рабочее состояние → Отмененная: задача отменена согласно процессу, если такой статус предусмотрен.

### Ограничения
- Монтажник может иметь только одну задачу в состоянии `В работе`.
- Если задача назначена конкретному сотруднику, другой сотрудник не может перевести ее в работу.

## Этап

### Состояния
- Не начат.
- В работе.
- Подтверждение.
- Завершен.
- Невозможно выполнить.

### Переходы
- Не начат → В работе: сотрудник открыл выполнение этапа.
- В работе → Подтверждение: сотрудник завершил действие и перешел к подтверждению.
- Подтверждение → Завершен: фото и данные подтверждения сохранены, если требуются.
- В работе → Невозможно выполнить: сотрудник зафиксировал причину невозможности.
- Подтверждение → В работе: ошибка фото или валидации требует повторить действие.

### Ограничения
- Завершение этапа должно быть связано с задачей, сотрудником и временем.
- Если фото требуется, этап не считается завершенным до успешной фиксации.

## Фото

### Состояния
- Не требуется.
- Ожидается.
- Снимается.
- Загружается.
- Загружено.
- Ошибка загрузки.
- Отклонено валидацией.

### Переходы
- Ожидается → Снимается: пользователь открывает камеру.
- Снимается → Загружается: фото сделано и отправляется.
- Загружается → Загружено: файл сохранен в storage и метаданные записаны.
- Загружается → Ошибка загрузки: сеть или сервер вернули ошибку.
- Загружено → Отклонено валидацией: файл не соответствует требованиям сценария.
- Ошибка загрузки → Загружается: повторная отправка.

### Ограничения
- Фото должно быть связано с действием, автором, временем и сущностью.
- Фото не заменяет бизнес-событие, а подтверждает его.

## Монеты

### Состояния
- Не рассчитаны.
- Рассчитываются.
- Начислены.
- Отображены.
- Ошибка расчета.

### Переходы
- Не рассчитаны → Рассчитываются: подтвержденное действие в статусе `На работе`.
- Рассчитываются → Начислены: backend применил правило начисления.
- Начислены → Отображены: UI получил обновленный баланс.
- Рассчитываются → Ошибка расчета: нарушение данных или техническая ошибка.

### Ограничения
- Монеты не начисляются в статусе `Ушел`.
- Прораб и финансист не начисляют монеты вручную.
- Историческое начисление должно оставаться проверяемым.

## ИИ

### Состояния
- Нет данных.
- Ожидает расчета.
- Анализирует.
- Рекомендация готова.
- Ошибка анализа.
- Рекомендация просмотрена.
- Решение принято человеком.

### Переходы
- Нет данных → Ожидает расчета: появились подтвержденные данные.
- Ожидает расчета → Анализирует: backend передал данные в AI Engine.
- Анализирует → Рекомендация готова: AI сформировал рекомендацию и объяснение.
- Анализирует → Ошибка анализа: AI не смог построить результат.
- Рекомендация готова → Рекомендация просмотрена: финансист открыл рекомендацию.
- Рекомендация просмотрена → Решение принято человеком: финансист подтвердил, отклонил или отложил решение в рамках процесса.

### Ограничения
- ИИ не переводит выплату в утвержденное состояние.
- ИИ не меняет начисленные монеты.
- Рекомендация должна быть объяснимой.

## Уведомления

### Состояния
- Новое.
- Просмотрено.
- Обработано.
- Скрыто.
- Ошибка обработки.

### Переходы
- Новое → Просмотрено: пользователь открыл уведомление.
- Просмотрено → Обработано: пользователь выполнил связанное действие или отметил обработку.
- Новое или Просмотрено → Скрыто: уведомление скрыто по разрешенному действию.
- Любое активное состояние → Ошибка обработки: API не смог сохранить изменение статуса.

### Ограничения
- Уведомление не выполняет финансовое действие самостоятельно.
- Связанная сущность должна быть доступна роли пользователя.

## Выплата

### Состояния
- Черновик.
- Готова к решению.
- Сохраняется.
- Утверждена финансистом.
- Отклонена.
- Отложена.
- Конфликт.
- Ошибка сохранения.

### Переходы
- Черновик → Готова к решению: данные начислений и AI-рекомендации загружены.
- Готова к решению → Сохраняется: финансист подтвердил действие.
- Сохраняется → Утверждена финансистом: backend сохранил историю выплаты.
- Готова к решению → Отклонена: финансист отклонил выплату, если процесс это допускает.
- Готова к решению → Отложена: финансист отложил решение, если процесс это допускает.
- Сохраняется → Конфликт: выплата за период уже создана.
- Сохраняется → Ошибка сохранения: техническая ошибка или ошибка валидации.

### Ограничения
- Повторная выплата за тот же период и объект не допускается.
- Итоговая сумма равна начисленным монетам плюс премия.
- Финальное решение принимает финансист.
