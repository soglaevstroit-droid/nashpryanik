# 002 — Modules

## Назначение
Документ описывает модули проекта, их назначение, ответственность, зависимости и используемые данные. Модули отражают текущую бизнес-логику и UI Bible, не добавляя новых ролей и сценариев.

## Авторизация

### Назначение
Определяет пользователя, его роль и доступные действия.

### Ответственность
- вход в систему;
- хранение текущей сессии;
- проверка роли на UI и backend;
- запрет недоступных действий.

### Зависимости
- Пользователи;
- Роли;
- API;
- Навигация.

### Используемые данные
- `userId`;
- роль;
- статус сессии;
- права доступа.

## Пользователи

### Назначение
Хранит сотрудников системы и их ролевую принадлежность.

### Ответственность
- профиль сотрудника;
- имя и фото;
- роль;
- связь с объектами, задачами, сменами и выплатами.

### Зависимости
- Авторизация;
- Объекты;
- Задачи;
- Рабочие смены;
- Финансы.

### Используемые данные
- сотрудник;
- роль;
- фото профиля;
- текущий статус;
- история активности.

## Объекты

### Назначение
Связывает задачи, сотрудников, аналитику и финансовые данные со строительным объектом.

### Ответственность
- фильтрация задач и аналитики;
- группировка сотрудников;
- историческая привязка финансовых данных;
- отчеты по объектам.

### Зависимости
- Пользователи;
- Задачи;
- Аналитика;
- Финансы.

### Используемые данные
- `objectId`;
- название объекта;
- задачи объекта;
- сотрудники объекта;
- финансовые показатели.

## Задачи

### Назначение
Основная единица работы на объекте.

### Ответственность
- создание задачи прорабом;
- назначение исполнителей;
- доступность для монтажников;
- переход в работу;
- завершение;
- связь с этапами, фото, монетами и историей.

### Зависимости
- Пользователи;
- Объекты;
- Этапы;
- Фото;
- Монеты;
- Уведомления.

### Используемые данные
- название;
- описание;
- объект;
- исполнители;
- статус;
- срок;
- текущий исполнитель;
- история событий.

## Этапы

### Назначение
Делят задачу на последовательные рабочие шаги.

### Ответственность
- хранение названия и описания;
- хранение фото-примера, если он есть;
- фиксация выполнения;
- поддержка статуса `Невозможно выполнить`;
- связь с фото подтверждения.

### Зависимости
- Задачи;
- Фото;
- Рабочая смена;
- Монеты.

### Используемые данные
- `stepId`;
- `taskId`;
- название;
- описание;
- фото-пример;
- статус;
- время выполнения.

## Фото

### Назначение
Подтверждает ключевые действия и формирует доказательную историю.

### Ответственность
- загрузка фото;
- хранение файла;
- сохранение метаданных;
- связь с задачей, этапом, сменой или событием;
- обработка состояний загрузки и ошибок.

### Зависимости
- API;
- Backend;
- File Storage;
- Задачи;
- Этапы;
- История.

### Используемые данные
- файл;
- ссылка на файл;
- автор;
- время;
- связанная сущность;
- статус загрузки.

## Монеты

### Назначение
Внутренняя цифровая единица оценки подтвержденного труда.

### Ответственность
- начисление только в статусе `На работе`;
- связь начисления с задачей, этапом и сотрудником;
- показ баланса;
- передача данных в аналитику, AI и финансы;
- защита от ручного изменения прорабом или финансистом.

### Зависимости
- Рабочая смена;
- Задачи;
- Этапы;
- Фото;
- Аналитика;
- Финансы.

### Используемые данные
- `employeeId`;
- `taskId`;
- `stepId`;
- количество монет;
- событие начисления;
- время начисления.

## Аналитика

### Назначение
Показывает факты, динамику, эффективность и отклонения.

### Ответственность
- общая сумма монет;
- рабочее время;
- динамика по дням, объектам и сотрудникам;
- диаграмма Ганта;
- эффективность;
- финансовые отчеты;
- обработка неполных данных.

### Зависимости
- Пользователи;
- Объекты;
- Задачи;
- Этапы;
- Рабочие смены;
- Монеты;
- Выплаты;
- AI.

### Используемые данные
- события;
- агрегаты;
- периоды;
- фильтры;
- отклонения;
- история.

## Финансы

### Назначение
Поддерживает финансовые экраны, премии, выплаты, историю и отчеты.

### Ответственность
- показ начисленных монет;
- показ рекомендаций ИИ;
- расчет итоговой суммы;
- сохранение решения финансиста;
- защита от повторной выплаты;
- история выплат;
- отчеты.

### Зависимости
- Монеты;
- Аналитика;
- AI;
- Пользователи;
- Объекты;
- Уведомления.

### Используемые данные
- начисленные монеты;
- AI-рекомендация;
- премия;
- комментарий;
- итог;
- утверждающий;
- дата выплаты.

## AI

### Назначение
Формирует рекомендации и объяснения на основе подтвержденных данных.

### Ответственность
- сравнение начисленных монет с AI-оценкой;
- рекомендации премий;
- объяснение отклонений;
- сигналы о подозрительных действиях;
- поддержка уведомлений.

### Зависимости
- Аналитика;
- Монеты;
- Задачи;
- Этапы;
- Рабочие смены;
- Финансы.

### Используемые данные
- выполненные задачи;
- рабочее время;
- монеты;
- история активности;
- объект;
- контекст выполнения;
- объяснение рекомендации.

## Настройки

### Назначение
Хранит настройки финансового раздела без изменения утвержденной бизнес-логики.

### Ответственность
- период аналитики по умолчанию;
- отображение монет;
- пороги отклонений;
- правила премий в допустимых границах;
- уведомления;
- фильтры объектов;
- режим только просмотр.

### Зависимости
- Пользователи;
- Финансы;
- Аналитика;
- Уведомления.

### Используемые данные
- настройки пользователя;
- доступные варианты;
- фильтры;
- пороги;
- режим просмотра.

## Уведомления

### Назначение
Сообщают о событиях, требующих внимания.

### Ответственность
- новые AI-рекомендации;
- большие отклонения;
- сотрудники без активности;
- подозрительные действия;
- готовые выплаты;
- события проверки;
- статусы просмотра и обработки.

### Зависимости
- AI;
- Аналитика;
- Финансы;
- Пользователи;
- Объекты.

### Используемые данные
- тип уведомления;
- статус;
- приоритет;
- связанная сущность;
- сотрудник;
- объект;
- время создания.

## История и аудит

### Назначение
Обеспечивает прозрачность действий системы.

### Ответственность
- фиксация значимых событий;
- связь события с автором, ролью и временем;
- восстановление истории задачи, смены, фото, монет, рекомендации и выплаты.

### Зависимости
- Все доменные модули.

### Используемые данные
- событие;
- автор;
- роль;
- время;
- связанная сущность;
- старое и новое состояние, если применимо.
