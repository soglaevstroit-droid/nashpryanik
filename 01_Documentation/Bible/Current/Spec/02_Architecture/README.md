# 02 Architecture

## Назначение
Раздел `02_Architecture` описывает архитектуру НАШПРЯНИК.РФ как рабочей системы для строительной компании. Документы объясняют, как мобильный интерфейс, backend, API, база данных, файловое хранилище, аналитика, AI Engine, уведомления и роли пользователей взаимодействуют между собой.

Архитектурная документация не вводит новую бизнес-логику. Она фиксирует техническую организацию уже утвержденных правил из `00_Project`, UI Bible и Design System.

## Структура раздела
- `001_SYSTEM_ARCHITECTURE.md` — общая архитектура системы и взаимодействие компонентов.
- `002_MODULES.md` — модули проекта, их ответственность, зависимости и используемые данные.
- `003_NAVIGATION.md` — навигация между экранами для монтажника, прораба и финансиста.
- `004_STATE_MACHINE.md` — состояния пользователя, смены, задачи, этапа, фото, монет, ИИ и уведомлений.
- `005_APP_LIFECYCLE.md` — жизненный цикл приложения от запуска до завершения рабочей смены и финансовых действий.
- `006_DATA_FLOW.md` — движение данных между UI, API, backend, storage, AI, analytics и finance.
- `007_EVENT_FLOW.md` — события системы и их обработка.
- `008_SCALABILITY.md` — принципы масштабирования на сотрудников, объекты, компании, роли, фото и модули.

## Источники истины
Архитектура должна соответствовать:
- `00_Project/001_PROJECT_VISION.md`;
- `00_Project/002_CORE_PRINCIPLES.md`;
- `00_Project/003_PRODUCT_PHILOSOPHY.md`;
- `00_Project/004_USER_ROLES.md`;
- `00_Project/005_BUSINESS_RULES.md`;
- `00_Project/006_SUCCESS_METRICS.md`;
- `00_Project/007_GLOSSARY.md`;
- UI Bible;
- Design System.

## Правила использования
1. Перед реализацией нового модуля проверить `002_MODULES.md`.
2. Перед созданием переходов проверить `003_NAVIGATION.md` и соответствующие `NAVIGATION.md` в UI Bible.
3. Перед изменением статусов проверить `004_STATE_MACHINE.md`.
4. Перед изменением загрузки фото, монет, AI-рекомендаций или выплат проверить `006_DATA_FLOW.md` и `007_EVENT_FLOW.md`.
5. Перед расширением проекта проверить `008_SCALABILITY.md`.

## Неприкосновенные ограничения
- ИИ рекомендует, человек утверждает.
- Монеты начисляются только в статусе `На работе`.
- Все ключевые действия монтажника требуют фото.
- Монтажник может выполнять только одну задачу одновременно.
- Прораб не начисляет монеты вручную.
- Финансист не редактирует начисленные монеты вручную.
- Финансовые решения сохраняются в истории.
- Утвержденный дизайн сохраняется минимум на 95%.
