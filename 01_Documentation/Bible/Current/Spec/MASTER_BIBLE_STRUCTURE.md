# Правило структуры

Фундамент определяет продукт.

Продукт определяет архитектуру.

Архитектура определяет код.

Код не должен противоречить фундаменту.

---

# Главное правило

Если новый документ не имеет места в этой структуре, значит:

1. структура требует обновления;

или

2. документ не нужен.

---

# Следующий порядок работы

1. Завершить `08_Process_Bible`.
2. Завершить `09_Event_Bible`.
3. Создать `07_Role_Bible`.
4. Создать `10_Knowledge_Bible`.
5. После этого переходить к инженерной реализации.
