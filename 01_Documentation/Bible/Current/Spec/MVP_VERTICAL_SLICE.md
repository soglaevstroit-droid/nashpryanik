Версия: 1.0  
Статус: APPROVED

---

# Цель документа

Описать первый полностью рабочий сценарий MVP.

Этот документ определяет минимальный вертикальный срез системы, который должен быть реализован первым.

Главный принцип:

> Лучше один сценарий, который работает от начала до конца, чем десять незавершенных функций.

---

# Главная цель MVP

Проверить, может ли система СТРОИТ.РФ обеспечить полный рабочий цикл:

монтажник → задача → этап → фото → история → монеты → контроль прораба → проверка финансиста.

---

# Участники сценария

## Монтажник

Выполняет задачу и фиксирует результат.

## Прораб

Создает задачу, контролирует выполнение и видит прогресс.

## Финансист

Видит подтвержденные работы и начисленные монеты.

## AI

Может анализировать данные, но не принимает решений.

---

# Вертикальный сценарий

1. Монтажник входит в приложение.
2. Система определяет его роль.
3. Монтажник нажимает «На работе».
4. Делает фото прихода.
5. Видит список доступных задач.
6. Открывает задачу.
7. Нажимает «Взять задачу».
8. Система проверяет, что у монтажника нет другой активной задачи.
9. Задача переходит в статус «В работе».
10. Монтажник открывает первый этап.
11. Читает описание этапа.
12. Видит фото-пример от прораба.
13. Выполняет работу.
14. Делает фото результата.
15. Нажимает «Завершить этап».
16. Фото загружается на сервер.
17. Событие записывается в историю.
18. Этап получает статус «Выполнен».
19. Система начисляет монеты.
20. Прораб видит обновление.
21. Финансист видит начисление.
22. Монтажник видит обновленный баланс.
23. В конце дня монтажник нажимает «Ушел».
24. Делает фото ухода.
25. Рабочая смена закрывается.

# Что обязательно должно работать

## Backend

- Авторизация.
- Проверка ролей.
- Получение списка задач.
- Взятие задачи.
- Получение этапов.
- Загрузка фото.
- Завершение этапа.
- Начисление монет.
- История действий.

- users
- work_sessions
- tasks
- task_steps
- photos
- coin_transactions
- audit_log

## Mobile

- экран входа;
- главный экран монтажника;
- список задач;
- карточка задачи;
- экран этапа;
- камера;
- экран результата;
- профиль / баланс.

## Прораб

- создание задачи;
- создание этапа;
- фото-пример;
- просмотр статуса выполнения.

## Финансист

- просмотр начисленных монет;
- просмотр истории;
- подтвержденные выполненные этапы.

# Что НЕ входит в первый вертикальный срез

- сложная аналитика;
- полноценный AI;
- магазин монет;
- рейтинги;
- сложные отчеты;
- интеграции с бухгалтерией;
- несколько компаний;
- сложная админка;
- web-версия для заказчика.

# Тестовый объект

Объект:

Пряник

Этаж:

3

Помещение:

323

Задача:

По существующей разметке прорезать штробы, удалить бетон, очистить каналы и убрать строительный мусор.

Этапы:

1. Сделать фото ДО.
2. Прорезать штробы по разметке.
3. Удалить бетон.
4. Очистить каналы.
5. Убрать строительный мусор.
6. Сделать фото результата.

---

# Критерии готовности

MVP Vertical Slice считается готовым, если:

- монтажник может войти;
- монтажник может начать смену;
- монтажник видит задачу;
- монтажник может взять задачу;
- монтажник может открыть этап;
- монтажник может сделать фото;
- фото сохраняется;
- этап завершается;
- история обновляется;
- монеты начисляются;
- прораб видит прогресс;
- финансист видит начисление;
- монтажник может завершить смену.

---

# Главный критерий

Если на объекте Пряник один монтажник может выполнить одну реальную задачу через приложение от начала до конца — первый MVP считается жизнеспособным.
