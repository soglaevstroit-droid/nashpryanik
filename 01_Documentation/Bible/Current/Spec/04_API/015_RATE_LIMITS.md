# 015 — Rate Limits

## Назначение
Rate limits защищают API от перегрузки и случайных повторов действий без изменения бизнес-логики.

## Общие лимиты
- `GET` экранных данных: до 120 запросов в минуту на пользователя.
- `POST` доменных действий: до 30 запросов в минуту на пользователя.
- Auth login: до 10 попыток в 10 минут на идентификатор.
- Загрузка фото: до 60 загрузок в час на пользователя.
- Экспорт отчетов: до 10 выгрузок в час на пользователя.

## Ответ при лимите

```json
{
  "error": {
    "code": "RATE_LIMITED",
    "message": "Слишком много запросов. Повторите позже.",
    "details": {
      "retryAfterSeconds": 60
    },
    "requestId": "req_rate_limit_001"
  }
}
```

## Заголовки

```http
X-RateLimit-Limit: 120
X-RateLimit-Remaining: 42
Retry-After: 60
```

## Особые правила
- Лимит не должен создавать дубли смен, задач, этапов или выплат.
- Повторная отправка фото после ошибки разрешена в рамках лимита.
- Создание выплаты защищается не только лимитом, но и проверкой `PAYMENT_CONFLICT`.
- AI-запросы могут кэшироваться backend, чтобы не пересчитывать одинаковую рекомендацию без причины.

## Связь с UI
UI показывает состояние `Disabled action` или объяснимую ошибку, если действие временно ограничено.
