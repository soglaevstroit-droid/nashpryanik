# Проверить согласованность

## Project

Проверить:

- роли

- терминологию

- определения

- бизнес-правила

- KPI

---

## UI Bible

Проверить:

- все экраны существуют

- отсутствуют дубликаты

- роли совпадают

- переходы совпадают

- используются компоненты Design System

---

## Architecture

Проверить:

- модули

- data flow

- state machine

- navigation

- event flow

Нет ли противоречий.

---

## Database

Проверить:

- все сущности используются

- нет лишних таблиц

- связи совпадают

- enum совпадают

- audit log соответствует API

---

## API

Проверить:

- каждый endpoint использует существующие таблицы

- нет ссылок на отсутствующие поля

- совпадают request/response

- права доступа соответствуют ролям

---

## Backend

Проверить:

- сервисы используют существующие репозитории

- контроллеры соответствуют API

- middleware используются

- фоновые задачи соответствуют бизнес-логике

---

## Frontend

Проверить:

- каждый экран существует

- навигация соответствует UI Bible

- API соответствует Backend

- состояния соответствуют API

---

## Design System

Проверить:

- используются одинаковые цвета

- одинаковые кнопки

- одинаковые статусы

- одинаковые карточки
