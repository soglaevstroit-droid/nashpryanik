# 00 Project

## Назначение
Раздел `00_Project` фиксирует фундаментальные правила продукта НАШПРЯНИК.РФ. Эти документы объясняют, зачем существует система, какие роли она поддерживает, какие бизнес-правила нельзя нарушать и по каким критериям принимаются продуктовые и инженерные решения.

Раздел используется как верхний уровень Developer Bible. Если экран, API, база данных, AI-логика или frontend-решение противоречат документам этого раздела, решение должно быть пересмотрено до реализации.

## Документы раздела
- `001_PROJECT_VISION.md` — миссия, долгосрочное видение, проблемы отрасли, решение и критерии принятия решений.
- `002_CORE_PRINCIPLES.md` — основные принципы проекта: простота, прозрачность, фотофиксация, единый источник информации, роль ИИ и сохранение утвержденного дизайна.
- `003_PRODUCT_PHILOSOPHY.md` — философия продукта для монтажника, прораба, финансиста и руководителя.
- `004_USER_ROLES.md` — обязанности, права, ограничения, экраны и основные действия ролей.
- `005_BUSINESS_RULES.md` — жизненный цикл задачи, рабочая смена, статусы, монеты, рекомендации ИИ, премии и ограничения системы.
- `006_SUCCESS_METRICS.md` — KPI продукта, сотрудников, интерфейса и качества.
- `007_GLOSSARY.md` — словарь терминов проекта.

## Как использовать раздел
1. Перед созданием нового экрана проверить `001_PROJECT_VISION.md`, `002_CORE_PRINCIPLES.md` и UI Bible.
2. Перед изменением бизнес-логики проверить `005_BUSINESS_RULES.md`.
3. Перед добавлением новой роли, права или ограничения проверить `004_USER_ROLES.md`.
4. Перед изменением визуального поведения проверить Design System и правило сохранения утвержденных референсов минимум на 95%.
5. Перед внедрением AI-функции проверить правило: ИИ рекомендует, человек утверждает.
6. Перед выпуском функциональности проверить KPI из `006_SUCCESS_METRICS.md`.

## Неприкосновенные правила
- Монтажник выполняет только одну задачу одновременно.
- Все ключевые действия монтажника подтверждаются фото.
- Монеты начисляются только в статусе `На работе`.
- Прораб создает задачи и этапы, но не начисляет монеты вручную.
- Финансист видит монеты, аналитику, AI-оценку и премии.
- ИИ никогда не утверждает выплаты и не принимает финальное управленческое решение.
- Утвержденный дизайн сохраняется минимум на 95%.
