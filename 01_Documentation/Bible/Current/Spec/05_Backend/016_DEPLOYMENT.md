# 016 — Deployment

## Назначение
Документ описывает подготовку backend к деплою и требования к окружениям.

## Этапы деплоя
1. Установить зависимости.
2. Проверить конфигурацию.
3. Запустить миграции PostgreSQL.
4. Проверить доступ к storage.
5. Проверить JWT secrets.
6. Проверить AI endpoint, если AI включен.
7. Запустить backend.
8. Запустить background workers.
9. Проверить health endpoint.

## Health check

```http
GET /api/v1/health
```

Response:

```json
{
  "data": {
    "status": "ok",
    "database": "ok",
    "storage": "ok"
  }
}
```

## Окружения
Development допускает подробные debug-логи. Production скрывает stack trace от клиента и использует защищенные secrets.

## Миграции
Миграции должны быть обратимо спроектированы там, где это возможно. Миграции не должны терять исторические монеты, выплаты, audit log и фото-метаданные.

## Workers
Фоновые workers деплоятся вместе с backend или отдельным процессом. Они используют ту же PostgreSQL и storage-конфигурацию.

## Rollback
Rollback не должен удалять уже созданные финансовые решения. Если код откатывается, данные остаются совместимыми с предыдущей версией или требуют явно описанной миграции.
