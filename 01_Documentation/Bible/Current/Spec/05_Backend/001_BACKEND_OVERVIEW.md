# 001 — Backend Overview

## Назначение
Документ описывает backend-архитектуру проекта. Backend является центральным слоем бизнес-логики между мобильным UI, REST API, PostgreSQL, файловым хранилищем, AI Engine, уведомлениями и аналитикой.

## Архитектурный стиль
Backend строится как слоистое приложение:

```text
HTTP / API Layer
  ↓
Controllers
  ↓
Services
  ↓
Repositories
  ↓
PostgreSQL / File Storage / External AI
```

Контроллеры отвечают за HTTP-контракт. Сервисы отвечают за бизнес-сценарии. Репозитории отвечают за доступ к данным. Middleware отвечает за общие поперечные задачи: auth, validation, logging, errors, rate limit и request id.

## API Layer
API Layer реализует REST API `/api/v1` из API Bible.

Обязанности:
- принимать HTTP-запросы;
- извлекать JWT;
- применять middleware;
- вызывать контроллеры;
- возвращать единый формат успешных ответов и ошибок;
- поддерживать `multipart/form-data` для фото.

## Business Layer
Business Layer находится в сервисах. Он реализует утвержденные правила:
- монтажник может выполнять только одну задачу одновременно;
- монеты начисляются только в статусе `На работе`;
- ключевые действия требуют фото;
- прораб создает задачи и этапы, но не начисляет монеты вручную;
- финансист принимает финальное решение по выплатам;
- ИИ только рекомендует.

## Repository Layer
Repository Layer инкапсулирует PostgreSQL-запросы. Сервисы не должны собирать SQL вручную в контроллерах.

Репозитории работают с таблицами Database Bible:
- `users`;
- `objects`;
- `tasks`;
- `task_assignees`;
- `task_steps`;
- `work_sessions`;
- `photos`;
- `coin_transactions`;
- `notifications`;
- `ai_recommendations`;
- `audit_log`.

## Database
PostgreSQL является источником структурированных данных. Все доменные запросы фильтруются по `company_id` из JWT. Backend обязан сохранять историю значимых действий.

## File Storage
Файлы фото хранятся вне PostgreSQL. Backend сохраняет файл в storage, затем пишет метаданные в `photos`. Файл не должен быть единственным источником бизнес-смысла.

## AI Integration
Backend собирает подтвержденные данные и передает их в AI Engine. AI возвращает рекомендацию и объяснение. Backend сохраняет результат в `ai_recommendations`. AI не утверждает выплаты и не меняет монеты.

## Background Jobs
Фоновые задачи выполняют тяжелые или отложенные процессы:
- пересчет агрегатов;
- запуск AI-анализа;
- доставка уведомлений;
- очистка временных файлов;
- подготовка отчетов;
- резервные операции.

## Транзакции
Операции, меняющие несколько таблиц, выполняются транзакционно:
- завершение этапа с фото и монетами;
- взятие задачи;
- начало и завершение смены;
- создание выплаты;
- изменение статуса AI-рекомендации;
- обработка уведомления.

## Наблюдаемость
Каждый запрос получает `requestId`. Логи, ошибки и audit-записи должны связываться с request id, пользователем, ролью, компанией и сущностью.
