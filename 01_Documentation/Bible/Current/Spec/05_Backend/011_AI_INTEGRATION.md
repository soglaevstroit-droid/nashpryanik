# 011 — AI Integration

## Назначение
Документ описывает взаимодействие backend с AI Engine. AI помогает анализировать данные, но не принимает финальные решения.

## Что отправляется в AI
Backend отправляет только подтвержденные данные:
- сотрудник;
- объект;
- период;
- начисленные монеты;
- рабочее время;
- выполненные задачи;
- этапы;
- скорость закрытия этапов;
- история активности;
- контекст отклонений;
- предыдущие рекомендации, если они нужны для объяснимости.

Файлы фото не отправляются без отдельного описанного процесса. По умолчанию AI получает метаданные и связи.

## Request example

```json
{
  "employeeId": "8d32b7d6-8a0d-4d7c-a49d-f8a1a0aa6f41",
  "objectId": "2d899118-90d4-48cb-bccf-5881cf9637e7",
  "periodFrom": "2026-07-01",
  "periodTo": "2026-07-06",
  "paidCoins": 120,
  "workDurationMinutes": 540,
  "completedTasksCount": 3,
  "averageStepCloseDurationMinutes": 45
}
```

## Что возвращается

```json
{
  "aiEstimatedCoins": 150,
  "difference": 30,
  "recommendationType": "bonus_recommended",
  "confidence": 0.82,
  "explanation": {
    "summary": "Факт выполнения выше ожиданий.",
    "signals": [
      "closed_tasks_above_expected",
      "fast_step_completion"
    ]
  }
}
```

## Ограничения
- AI не утверждает выплаты.
- AI не начисляет монеты.
- AI не меняет статус задач.
- AI не скрывает уведомления.
- AI-рекомендация должна быть объяснимой.

## Сохранение результата
`AIService` сохраняет результат в `ai_recommendations` через `AIRecommendationRepository`.

Если рекомендация требует внимания, `NotificationService` создает уведомление.

## Ошибки
Ошибки AI возвращаются как:
- `AI_ANALYSIS_ERROR`;
- `INTERNAL_ERROR`, если сбой инфраструктурный.

Ошибка AI не должна ломать рабочий сценарий монтажника или прораба.

## Audit
Создание рекомендации, просмотр и фиксация решения человека пишутся в `audit_log`.
