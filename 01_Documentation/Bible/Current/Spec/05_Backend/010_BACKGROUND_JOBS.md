# 010 — Background Jobs

## Назначение
Фоновые задачи выполняют тяжелые, отложенные и повторяемые процессы без блокировки HTTP-запроса.

## Принципы
- HTTP-запрос должен завершаться быстро.
- Job должен быть идемпотентным.
- Job должен иметь request или correlation id.
- Ошибка job логируется и не создает скрытое финальное бизнес-состояние.

## Пересчет монет и агрегатов
Монеты начисляются в транзакции доменного события. Фоновые задачи могут пересчитывать агрегаты для аналитики:
- суммы по дням;
- суммы по объектам;
- суммы по сотрудникам;
- рабочее время.

Агрегаты воспроизводимы из `coin_transactions` и `work_sessions`.

## AI-анализ
AI job:
1. Получает событие или расписание.
2. Собирает подтвержденные данные.
3. Вызывает AI Engine.
4. Сохраняет рекомендацию.
5. Создает уведомление для финансиста, если требуется.

AI job не утверждает выплату.

## Отправка уведомлений
Notification job доставляет:
- in-app уведомления;
- push, если канал включен;
- email, если канал включен.

Статус доставки не заменяет статус обработки уведомления пользователем.

## Очистка временных файлов
Job удаляет:
- временные uploads старше допустимого времени;
- orphan storage files без записи `photos`;
- неиспользуемые thumbnails, если они безопасно восстанавливаемы.

## Резервное копирование
Backup job должен учитывать:
- PostgreSQL;
- файловое хранилище;
- соответствие `photos.storage_key`;
- audit log;
- финансовую историю.

## Экспорт отчетов
Финансовые отчеты могут готовиться job, если запрос тяжелый. UI получает статус подготовки и ссылку на готовый файл.

## Retry
Для временных ошибок используется retry с ограничением. Для бизнес-ошибок retry не применяется без изменения входных данных.

## Наблюдаемость
Каждый job логирует:
- имя;
- параметры без секретов;
- company id;
- статус;
- длительность;
- ошибку, если есть.
