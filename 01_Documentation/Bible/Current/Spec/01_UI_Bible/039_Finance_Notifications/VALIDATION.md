# VALIDATION — 39 Уведомления финансиста

## Проверки
- `notificationId` обязателен для действий с уведомлением.
- Тип уведомления должен быть одним из поддержанных.
- Статус уведомления должен быть одним из поддержанных.
- `periodFrom` не может быть позже `periodTo`.
- `objectId` должен существовать, если передан.
- При связанном сотруднике `employeeId` должен существовать.
- Связанная сущность должна быть доступна роли финансиста.

## Отображение
- Новые уведомления должны визуально отличаться от просмотренных.
- Приоритет отображается средствами Design System.
- Уведомление без связанной сущности остается доступным для просмотра.
