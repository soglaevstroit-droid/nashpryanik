# VALIDATION — 31 Аналитика сотрудника

## Проверки
- `employeeId` обязателен.
- Сотрудник должен быть доступен роли финансиста.
- `periodFrom` не может быть позже `periodTo`.
- Монеты сотрудника не могут быть отрицательными.
- Рабочее время не может быть отрицательным.
- Количество выполненных задач не может быть отрицательным.
- События истории должны иметь дату и тип активности.
- Сумма монет в истории должна соответствовать итоговой сумме за период.

## Отображение
- Монеты отображаются целым числом.
- Рабочее время отображается в часах и минутах.
- При отсутствии активности показывается пустое состояние, а не ошибка.
