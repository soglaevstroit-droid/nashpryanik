# VALIDATION — 40 Настройки финансиста

## Проверки
- Период аналитики по умолчанию должен быть одним из поддержанных вариантов.
- Режим отображения монет должен быть одним из поддержанных вариантов.
- Пороги отклонений не могут быть отрицательными.
- Минимальный порог не может быть больше максимального.
- Правила премий должны соответствовать бизнес-логике проекта.
- Настройки уведомлений должны содержать только поддержанные типы событий.
- Фильтры объектов должны ссылаться на существующие объекты.
- Режим только просмотр должен быть булевым значением.
- Нельзя сохранить настройку автоматического утверждения выплат ИИ.

## Отображение
- Ошибки показываются рядом с соответствующим блоком.
- Несохраненные изменения должны быть заметны пользователю.
- После сохранения показывается подтверждение.
