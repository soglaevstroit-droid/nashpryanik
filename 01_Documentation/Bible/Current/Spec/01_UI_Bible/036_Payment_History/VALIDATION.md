# VALIDATION — 36 История выплат

## Проверки
- `periodFrom` не может быть позже `periodTo`.
- `objectId` должен существовать, если передан.
- `employeeId` должен существовать, если передан.
- Начислено не может быть отрицательным.
- Рекомендация ИИ не может быть отрицательной.
- Премия не может быть отрицательной.
- Итог должен равняться начисленным монетам плюс премия.
- `approvedBy` обязателен для утвержденной выплаты.
- `paidAt` обязателен для записи истории выплаты.

## Отображение
- Суммы отображаются целым числом.
- Период отображается в понятном диапазоне дат.
- При отсутствии фильтрованных записей показывается пустое состояние.
