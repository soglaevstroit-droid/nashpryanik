# LOGIC — 36 История выплат

## Основная логика
Экран показывает журнал выплат, утвержденных финансистом.

## Отображаемые данные
- Сотрудник.
- Объект.
- Период.
- Начислено.
- Рекомендация ИИ.
- Премия.
- Итог.
- Кто утвердил.
- Дата выплаты.

## Фильтры
- Фильтр по объекту ограничивает выплаты выбранным объектом.
- Фильтр по сотруднику показывает выплаты выбранного сотрудника.
- Фильтр по периоду показывает выплаты, дата или расчетный период которых пересекается с выбранным диапазоном.

## Историчность
- История хранит значения на момент утверждения выплаты.
- Если сотрудник, объект или рекомендация изменились позже, запись истории не пересчитывается автоматически.
- Запись истории не меняет начисленные монеты задним числом.

## Правила финансиста
- Финансист просматривает и сверяет историю выплат.
- Финансист не редактирует утвержденные выплаты на этом экране.
- Новая выплата создается на экране выплат.
