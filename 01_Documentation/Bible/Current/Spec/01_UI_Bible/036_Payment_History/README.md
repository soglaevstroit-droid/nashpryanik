# 36 — История выплат

## Назначение
Экран истории выплат с данными по сотруднику, объекту, периоду, начислениям, AI рекомендации, премии, итогу, утверждающему и дате выплаты.

## Роль
Финансист / выплаты.

## Статус
APPROVED REFERENCE REQUIRED.

## Требование
Сохранить визуальный стиль утвержденных референсов минимум на 95%.

## Главное правило
История выплат является журналом принятых решений и не редактирует прошлые начисления.
