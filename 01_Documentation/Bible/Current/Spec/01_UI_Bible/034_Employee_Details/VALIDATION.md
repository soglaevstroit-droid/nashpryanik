# VALIDATION — 34 Карточка сотрудника

## Проверки
- `employeeId` обязателен.
- Сотрудник должен быть доступен роли финансиста.
- `periodFrom` не может быть позже `periodTo`.
- Рабочее время не может быть отрицательным.
- Заработанные монеты не могут быть отрицательными.
- Количество выполненных задач не может быть отрицательным.
- Текущая задача должна иметь `taskId`, если она отображается.
- События календаря должны иметь дату.
- События истории должны иметь тип и дату.

## Отображение
- Фото сотрудника должно иметь безопасное fallback-состояние.
- Монеты отображаются целым числом.
- Рабочее время отображается в часах и минутах.
- Пустые блоки показывают нейтральное состояние, а не ошибку.
