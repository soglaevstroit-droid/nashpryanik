# VALIDATION — 37 Аналитика эффективности

## Проверки
- `periodFrom` не может быть позже `periodTo`.
- `employeeId` должен существовать, если передан.
- `objectId` должен существовать, если передан.
- Монеты не могут быть отрицательными.
- Рабочее время не может быть отрицательным.
- Количество закрытых задач не может быть отрицательным.
- Скорость закрытия этапов не может быть отрицательной.
- Отклонение должно рассчитываться из факта и ожиданий.

## Отображение
- Монеты отображаются целым числом.
- Рабочее время отображается в часах и минутах.
- Отсутствующие ожидания показываются как неполные данные, а не как ошибка.
