## Результаты проверки

### Терминология

Терминология проекта приведена к единому названию `НАШПРЯНИК.РФ`. Основные термины `монтажник`, `прораб`, `финансист`, `руководительская роль`, `объект`, `задача`, `этап`, `смена`, `фото`, `монеты`, `AI-рекомендация`, `выплата` и `аудит` используются согласованно.

### Роли

Согласованные роли:

- `worker` — монтажник;
- `foreman` — прораб;
- `finance` — финансист;
- `manager` — руководительская роль без отдельного набора MVP-экранов.

Отдельная роль администратора не введена в продуктовую модель. Frontend-документация корректно фиксирует, что admin UI, маршруты и права не реализуются до появления такой роли в фундаментальной документации.

### Статусы

Статусы смен, задач, этапов, фото, уведомлений и AI-рекомендаций согласованы между UI Bible, Database Bible, API Bible и Backend Bible. Неподтвержденный тип операции с монетами удален, чтобы enum соответствовал утвержденной логике начисления.

### API и Database

REST API использует версионирование `/api/v1`, JWT-аутентификацию, RBAC и единый формат ошибок. Endpoint связаны с PostgreSQL-сущностями: пользователи, компании, объекты, задачи, этапы, смены, фото, монеты, уведомления, AI-рекомендации и аудит.

Финансовые endpoint описывают выплаты как решение финансиста. Database Bible фиксирует сущность выплаты как часть финансовой истории без введения нового сценария принятия решений.

### Backend и API

Backend Bible соответствует API Bible: контроллеры, сервисы, репозитории, middleware, ошибки, фоновые задачи, хранение файлов, AI-интеграция и RBAC описаны в тех же границах. ИИ не утверждает выплаты, не меняет монеты и не обходит решение человека.

### Frontend и API

Frontend Bible соответствует API Bible и Backend Bible: Flutter-приложение использует описанные endpoint, RBAC, единый формат ошибок, offline queue, локальное хранилище, загрузку фотографий и уведомления без расширения бизнес-логики.

### UI и Backend

UI-сценарии не изменялись. Backend-границы соответствуют утвержденным сценариям: монтажник выполняет задачи и подтверждает действия фото, прораб управляет задачами и сотрудниками, финансист анализирует монеты, отчеты, AI-рекомендации и принимает решения по выплатам.
