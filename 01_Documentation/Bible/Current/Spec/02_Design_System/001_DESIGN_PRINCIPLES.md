# НАШПРЯНИК.РФ — Design System
## 001 — Design Principles

Version: 1.0

## Назначение
Этот документ фиксирует базовые правила интерфейса. Любой экран должен сохранять утвержденный стиль референсов минимум на 95%.

## Главный принцип
Максимум информации. Минимум действий. Минимум текста.

## Правила
1. Один экран — одна главная задача.
2. На экране только одно главное действие.
3. Белый фон и много воздуха.
4. Основной акцент — фирменный оранжевый.
5. Монтажнику показываем только то, что помогает работать.
6. Прорабу показываем сотрудников, статусы и текущую работу.
7. Финансисту показываем цифры, факты, историю, ИИ-оценку и премии.
8. Все интерфейсы проектируются сначала под iPhone.

## Запрещено
- добавлять лишние блоки;
- менять стиль без согласования;
- использовать разные радиусы и стили карточек;
- показывать тариф монтажнику;
- дублировать одну и ту же шкалу прогресса;
- использовать разные иконки задач там, где утверждена одна фирменная иконка.

## Definition of Done
Экран готов, если он:
- повторяет утвержденный референс минимум на 95%;
- сохраняет бизнес-логику;
- не перегружен;
- удобен на телефоне;
- не требует устных объяснений.
