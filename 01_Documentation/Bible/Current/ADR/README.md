# Architecture Decision Records (ADR)

## Назначение

Данный раздел содержит все ключевые архитектурные решения проекта СТРОИТ.РФ.

Каждое решение фиксируется один раз и после утверждения становится частью инженерной документации проекта.

Цель ADR:

- сохранить историю решений;
- объяснить причины выбора;
- избежать повторного обсуждения уже принятых решений;
- ускорить адаптацию новых разработчиков;
- обеспечить единое понимание архитектуры.

---

## Статусы

- Proposed — предложено
- Approved — утверждено
- Superseded — заменено новым решением
- Deprecated — больше не используется

---

## Правила

1. Каждое важное архитектурное решение оформляется отдельным ADR.
2. Один ADR = одно решение.
3. После утверждения решение считается обязательным.
4. Изменение решения требует создания нового ADR.
5. Старые ADR никогда не удаляются.
