# Единый стандарт фотографий

Во всех пользовательских карточках СТРОИТ.РФ используется компонент `PhotoSlider` из
`apps/demo/public/photo-slider.js`. Собственные галереи, вертикальные наборы миниатюр и отдельные
варианты поведения для разных страниц не допускаются.

## Порядок содержимого карточки

1. Заголовок.
2. Краткое описание.
3. Большой `PhotoSlider`.
4. Статус, время и остальная информация.

Если у события истории нет фотографий, место под слайдер не резервируется. В задачах и деталях
используется понятное пустое состояние.

## Поведение

- Все слайды имеют единую фиксированную высоту компонента. Ширина вычисляется из исходного
  соотношения сторон изображения, поэтому фотография не растягивается и не сжимается.
- Следующая фотография видна справа, когда ширина текущего кадра меньше доступной области.
- Используются нативная горизонтальная прокрутка, CSS Scroll Snap и системная инерция.
- Точки показываются только при двух и более фотографиях; активная точка оранжевая.
- Первая и соседняя фотографии загружаются заранее, остальные — лениво по мере приближения.
- Двойной тап или двойной клик открывает полноэкранный просмотр без повторной загрузки.
- Полноэкранный просмотр поддерживает масштаб до 4x, pinch-to-zoom, перемещение, двойной тап,
  горизонтальное переключение и закрытие свайпом вниз или кнопкой.
- При повторной отрисовке карточки слайдер начинается с первого кадра.
- Все демонстрационные задачи, этапы и события содержат минимум две фотографии, чтобы поведение
  слайдера можно было проверить на любом экране.

## Использование

Компонент обязателен для задач, истории, деталей задачи и этапов. Он же должен использоваться в
будущих комментариях и отчётах.

Backend передаёт только безопасные метаданные артефакта и идентификатор для защищённой загрузки.
`storageKey` и параметры хранилища во frontend не передаются.

Текущее ограничение: локальный backend отдаёт исходный файл. Перед эксплуатацией больших альбомов
нужно добавить серверные производные размеры, сохранив идентичный API и поведение `PhotoSlider`.
