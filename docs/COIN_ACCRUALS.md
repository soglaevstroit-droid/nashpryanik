# Начисления за смену

Источник истины — backend. Финансовые значения хранятся в целых сотых монеты: 100 units равны
одной монете.

## Политика

- `COIN_UNITS_PER_SECOND = 21` — 0,21 монеты за целую секунду.
- `DAILY_STANDARD_LIMIT_COIN_UNITS = 700000` — максимум 7000 стандартных монет за смену.
- Активная сумма каждый раз пересчитывается из серверного `startedAt`, а не накапливается таймером.
- После достижения лимита последующие целые секунды сохраняются как переработка по той же ставке.
- При закрытии стандартные units округляются математически до целой монеты на backend.

## Состояния

- `ACTIVE` создаётся вместе со сменой.
- `PENDING_APPROVAL` создаётся при закрытии и входит только в «На согласовании».
- `APPROVED` входит в подтверждённый баланс.
- `REJECTED` не входит ни в pending, ни в approved.

Финансист подтверждает или отклоняет стандартную сумму. Повторное решение блокируется статусом.
Переработка не входит в pending автоматически: аналитик отдельно видит рассчитанную сумму, события и
фотографии смены, после чего подтверждает, корректирует или отклоняет её.

## API

- `GET /api/v1/worker/summary`
- `GET /api/v1/finance/accruals/pending`
- `PATCH /api/v1/finance/accruals/:id/approve`
- `PATCH /api/v1/finance/accruals/:id/reject`
- `GET /api/v1/analyst/accruals/overtime`
- `PATCH /api/v1/analyst/accruals/:id/overtime-review`

Worker не передаёт сумму, время или user id и не может выполнять операции согласования.
