# docs

Назначение: будущая инженерная документация, связанная с кодовой базой монорепозитория.

Главный источник истины проекта остается в `01_Documentation/Bible/Current/Spec`.

На Sprint 001 каталог создается как место для будущих технических runbook, ADR, onboarding notes и release материалов. Новые продуктовые правила, роли, статусы, Event Bible, Rule Bible или архитектурные решения здесь не создаются.
